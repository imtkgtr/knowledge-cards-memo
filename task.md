### タスク名: 知識キャンバス技術仕様書の作成
- 目的:
  基盤仕様を実装可能な粒度まで技術仕様へ落とし込み、採用技術・責務分割・実装順序を明確にする。
- 主な実装内容:
  `docs/` に統合技術仕様書を追加し、認証、キャンバス描画、状態管理、保存、添付、検索、整列、テスト、運用の候補技術と採否理由を機能別資料へ整理する。さらに、DB、API、フロントエンド、編集挙動、実装順序を実装着手可能な粒度の詳細仕様へ分解する。
- 完了条件:
  統合技術仕様書 1 本、機能別の技術調査資料群、実装用の詳細仕様書群が作成され、DB・API・画面・状態管理の各レイヤでそのまま実装に入れる判断材料が `docs/` 内で完結している。
- 備考:
  既存スタックである Next.js / FastAPI / Supabase を尊重しつつ、公式ドキュメントベースで採用候補を整理する。

- 目的:
  詳細仕様書に基づき、認証、キャンバス一覧、編集画面、カード操作、リンク、保存などの MVP 機能を段階的に実装し、実運用に近い形で操作確認できる状態へ進める。
- 主な実装内容:
  Supabase / FastAPI / Next.js の接続、キャンバス一覧 CRUD、キャンバス document API、React Flow ベースの編集画面、カード CRUD、リンク追加、ロック、複数選択、保存、必要なテストと記録更新を行う。
- 完了条件:
  主要な MVP フローであるログイン、キャンバス作成、キャンバス表示、カード編集、リンク編集、保存が通り、残課題が `progress.md` で追跡可能になっている。
- 備考:
  各機能は 1 コミット 1 目的で分割し、`progress.md` に未解決事項と次アクションを必ず残す。
